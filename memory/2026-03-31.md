## 💡 Technical Learnings

### Tempo Blockchain Integration
- **Chain ID**: 42431 (Moderato testnet), 111 (Tempo mainnet)
- **Native currency**: USD (stablecoin)
- **RPC**: `https://rpc.moderato.tempo.xyz`
- **Gas**: Paid in stablecoins, zero gas for passkey users
- **TIP-20**: Standard ERC-20 with fee AMM + DEX enshrined

### Wagmi Chain Configuration
- **Problem**: Custom chains with `as const` still trigger type errors
- **Solution**: Cast `as any` in `createConfig()` or use `wagmi/tempo` exports
- **Pattern**: Use `as any` temporarily, refactor when wagmi v3 releases

### Backend Security Best Practices
- **Rate limiting**: In-memory Map (prod: Redis)
- **CORS**: Dynamic via env variable, never `*` in production
- **API key**: Environment variable only, never in code
- **Private keys**: NEVER commit to version control (`.env` in `.gitignore`)

## 🎯 Product Decisions

### AgentPad = Product, Not Protocol
- **Constraint**: Minimal, deployable, no overengineering
- **Flow**: Token → 1 Pool → Multi-LP → Fee → Split
- **Eliminated**: Multiple pools, vaults, vesting, TWAP, hooks, adapters
- **Kept**: Core value (launch tokens + liquidify + earn fees)

### Security = First Priority
- **LP Ownership**: LpLockerv2 (not creator) → prevents rug-pull
- **Fee Split**: 20% ecosystem + 80% creator → transparent + fair
- **Passkey Storage**: Backend stores only public keys → private keys stay on device
